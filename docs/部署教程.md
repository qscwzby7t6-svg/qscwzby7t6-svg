## 1. Windows 本地部署

### 1.1 环境准备

#### 1.1.1 安装 Python 3.10+

1. 访问 [Python官网](https://www.python.org/downloads/windows/)
2. 下载 Python 3.10 或更高版本（建议 3.10.11）
3. **安装时勾选** "Add Python to PATH"
4. 安装完成后验证：
   ```cmd
   python --version
   pip --version
   ```

#### 1.1.2 安装 Git

1. 访问 [Git官网](https://git-scm.com/download/win)
2. 下载并安装 Git for Windows
3. 验证安装：
   ```cmd
   git --version
   ```

### 1.2 克隆项目

```cmd
git clone https://github.com/qscwzby7t6-svg/qscwzby7t6-svg.git
cd qscwzby7t6-svg
```

### 1.3 创建虚拟环境（推荐）

```cmd
# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
venv\Scripts\activate.bat

# 查看虚拟环境是否激活（命令行前会显示 (venv)）
```

### 1.4 安装依赖

```cmd
pip install -r requirements.txt
```

> **注意**: 如果安装 torch 失败，请使用以下命令：
> ```cmd
> pip3 install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118
> ```

### 1.5 运行应用

```cmd
# 开发模式
streamlit run app.py

# 生产模式（指定端口）
streamlit run app.py --server.port=8501
```

### 1.6 访问应用

打开浏览器访问：`http://localhost:8501`

## 2. 云服务器部署

### 2.1 环境准备（以 Ubuntu 22.04 为例）

#### 2.1.1 更新系统

```bash
sudo apt update && sudo apt upgrade -y
```

#### 2.1.2 安装依赖

```bash
# 安装 Python 和 Git
sudo apt install python3 python3-pip git -y

# 安装音频依赖
sudo apt install libsndfile1 ffmpeg -y
```

#### 2.1.3 安装虚拟环境工具

```bash
sudo apt install python3-venv -y
```

### 2.2 克隆项目

```bash
git clone https://github.com/qscwzby7t6-svg/qscwzby7t6-svg.git
cd qscwzby7t6-svg
```

### 2.3 创建虚拟环境

```bash
python3 -m venv venv
source venv/bin/activate
```

### 2.4 安装依赖

```bash
pip install -r requirements.txt
```

### 2.5 使用 systemd 配置服务（推荐）

创建服务文件 `/etc/systemd/system/ai-song-creator.service`：

```ini
[Unit]
Description=AI Song Creator Service
After=network.target

[Service]
User=www-data
Group=www-data
WorkingDirectory=/path/to/qscwzby7t6-svg
ExecStart=/path/to/qscwzby7t6-svg/venv/bin/streamlit run app.py --server.port=8501 --server.headless=true
Restart=always
Environment=PATH=/path/to/qscwzby7t6-svg/venv/bin

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
# 重新加载 systemd
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start ai-song-creator

# 设置开机自启
sudo systemctl enable ai-song-creator

# 查看服务状态
sudo systemctl status ai-song-creator
```

### 2.6 配置 Nginx 反向代理（可选）

安装 Nginx：

```bash
sudo apt install nginx -y
```

创建配置文件 `/etc/nginx/sites-available/ai-song-creator`：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:8501;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # WebSocket 支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/ai-song-creator /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### 2.7 配置 HTTPS（推荐）

使用 Certbot 配置 Let's Encrypt：

```bash
sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx -d your-domain.com
```

## 3. Aid Learning 部署 ⭐推荐

> **适用设备**: 荣耀平板、小米平板、华为平板等 Android 设备 + Aid Learning

### 3.1 环境准备

#### 3.1.1 检查系统

确保 Aid Learning 已正确安装并启动。

#### 3.1.2 打开终端

在 Aid Learning 界面中打开终端应用。

#### 3.1.3 更新系统包

```bash
# 更新 apt 包列表
apt update

# 升级已安装的包
apt upgrade -y
```

### 3.2 安装系统依赖（关键步骤）

这一步**非常重要**，缺少这些依赖会导致后续安装失败：

```bash
# 安装基础工具（git 必需！）
apt install git wget -y

# 安装编译工具链（用于从源码编译Python包）
apt install build-essential gcc g++ gfortran cmake -y

# 安装 Python 开发包（提供Python.h等头文件）
apt install python3-dev python3-pip -y

# 安装虚拟环境工具（创建Python虚拟环境必需）
apt install python3-venv -y

# 安装音频处理依赖（librosa、soundfile必需）
apt install libsndfile-dev ffmpeg -y

# 安装其他可能需要的库
apt install libblas-dev liblapack-dev libatlas-base-dev -y
```

> ⚠️ **注意**: 如果提示权限不足，使用 `sudo` 或联系 Aid Learning 管理员。

### 3.3 检查 Python 版本

```bash
# 查看 Python 版本
python3 --version

# 查看 pip 版本
pip3 --version

# 确保是 Python 3.10 或更高版本
```

### 3.4 克隆项目

```bash
# 创建项目目录
mkdir -p ~/projects
cd ~/projects

# 克隆项目
git clone https://github.com/qscwzby7t6-svg/qscwzby7t6-svg.git
cd qscwzby7t6-svg
```

> **注意**: 如果 git clone 失败，可能需要配置代理或使用 HTTPS：
> ```bash
> git config --global http.proxy ""
> git clone https://github.com/qscwzby7t6-svg/qscwzby7t6-svg.git
> ```

### 3.5 创建虚拟环境

```bash
# 创建虚拟环境
python3 -m venv venv

# 激活虚拟环境
source venv/bin/activate

# 验证激活成功（命令行前会显示 (venv)）
```

### 3.6 升级 pip（重要）

```bash
# 升级 pip 到最新版本
pip install --upgrade pip

# 设置 pip 使用国内镜像（加速下载）
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

### 3.7 安装依赖（分步安装）

为避免版本冲突，按照以下顺序安装：

```bash
# 第一步：安装基础包（先装numpy和scipy，避免后续包从源码编译）
# 这些包编译时间长，先安装预编译版本可以节省时间
pip install numpy==1.24.0 scipy==1.11.0

# 第二步：安装 PyTorch CPU 版本（推荐平板使用CPU版本）
pip install torch==2.0.0 torchaudio==2.0.0 --index-url https://download.pytorch.org/whl/cpu

# 第三步：安装 requirements.txt 中的其他依赖
# 注意：pip 不会降级已安装的包，所以 numpy/scipy/torch 保持已安装的版本
# requirements.txt 中的版本约束（>=1.24.0,<2.0.0）与手动安装的版本兼容
pip install -r requirements.txt

# 第四步：验证关键包版本（确保版本正确）
python -c "import numpy; import scipy; import torch; import librosa; import soundfile; print('所有依赖安装成功！')"
```

> 💡 **为什么这样安装？**
> 1. **numpy/scipy**：编译耗时，先装预编译版本避免从源码编译
> 2. **torch**：体积大，单独安装并使用CPU版本（平板通常无GPU）
> 3. **requirements.txt**：安装其他依赖，pip不会覆盖已安装的基础包
> 4. **版本兼容**：requirements.txt中的版本约束与手动安装的版本兼容

> ⚠️ **注意**: 如果某一步失败，查看错误信息单独解决，不要跳过步骤。

### 3.8 创建数据目录

## 5. 常见问题

### Q1: `python3-venv` 安装失败

**问题**: `Unable to locate package python3-venv`

**解决方案**:
```bash
# 更新包列表
apt update

# 如果还是不行，尝试安装
apt install python3-full -y

# 或使用 pip 安装 venv
pip install virtualenv
python -m virtualenv venv
```

### Q2: `libsndfile-dev` 安装失败

**问题**: `Unable to locate package libsndfile-dev`

**解决方案**:
```bash
# 先更新 apt
apt update

# 尝试安装（不同系统包名可能不同）
apt install libsndfile-dev -y

# 如果失败，尝试以下包名
apt install libsndfile1-dev -y

# 如果还是不行，使用 pip 安装预编译版本
pip install soundfile --no-cache-dir
```

### Q3: `scipy` 从源码安装失败

**问题**: `building 'scipy' extension` 失败

**解决方案**:
```bash
# 确保安装了所有编译工具
apt install build-essential gcc g++ gfortran cmake -y

# 确保安装了 BLAS 和 LAPACK
apt install libblas-dev liblapack-dev libatlas-base-dev -y

# 指定安装预编译版本
pip install scipy==1.11.0 --only-binary=:all:
```

### Q4: `torch` 安装失败

**问题**: PyTorch 下载或安装失败

**解决方案**:
```bash
# 使用 CPU 版本（推荐平板使用）
pip install torch==2.0.0 --index-url https://download.pytorch.org/whl/cpu

# 如果网络慢，使用国内镜像
pip install torch==2.0.0 -i https://pypi.tuna.tsinghua.edu.cn/simple

# 或跳过 torch（仅使用 CPU 功能）
pip install torch==2.0.0 --no-deps
```

### Q5: `librosa` 安装失败

**问题**: `No module named 'llvmlite'` 或 librosa 相关错误

**解决方案**:
```bash
# 先安装依赖
pip install numba numpy scipy

# 再安装 librosa
pip install librosa==0.10.0

# 如果还是失败，尝试最新版本
pip install librosa --upgrade
```

### Q6: 端口被占用

**问题**: `Address already in use`

**解决方案**:
```bash
# 查看端口占用
lsof -i :8501

# 杀死占用进程
kill -9 <PID>

# 或使用其他端口
streamlit run app.py --server.port=8502
```

### Q7: 存储空间不足

**问题**: 安装过程中提示空间不足

**解决方案**:
```bash
# 清理 apt 缓存
apt clean
apt autoremove -y

# 清理 pip 缓存
pip cache purge

# 清理临时文件
rm -rf /tmp/*

# 删除不必要的包
apt remove --purge -y $(dpkg -l | grep ^rc | awk '{print $2}')
```

### Q8: 内存不足

**问题**: 安装过程中被杀或提示 OOM

**解决方案**:
```bash
# 创建 swap 文件（推荐 2GB）
dd if=/dev/zero of=/swapfile bs=1M count=2048
mkswap /swapfile
swapon /swapfile

# 添加到 /etc/fstab 永久生效
echo '/swapfile none swap sw 0 0' >> /etc/fstab

# 限制 pip 使用的线程数
export OMP_NUM_THREADS=2
export MKL_NUM_THREADS=2
```

### Q9: 网络问题导致下载失败

**问题**: pip install 超时或下载失败

**解决方案**:
```bash
# 使用国内镜像
pip install xxx -i https://pypi.tuna.tsinghua.edu.cn/simple

# 或使用阿里云镜像
pip install xxx -i https://mirrors.aliyun.com/pypi/simple/

# 设置超时时间
pip install xxx --default-timeout=100
```

### Q10: 权限问题

**问题**: Permission denied

**解决方案**:
```bash
# 使用 --user 安装
pip install xxx --user

# 或使用 sudo（谨慎使用）
sudo pip install xxx

# 修改目录权限
chmod -R 755 ~/projects
```
