### 1.2 技术栈选择

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | Streamlit / Gradio | 快速构建数据科学应用界面 |
| 后端 | FastAPI | 高性能异步API框架 |
| 音频处理 | Librosa, PyDub | 音频加载、特征提取、格式转换 |
| 机器学习 | PyTorch | 深度学习模型框架 |
| 风格分析 | CCMusic Dashboard (VGG19/ResNet) | 开源音乐分类方案 |
| 音乐生成 | SongGeneration (腾讯开源) | 30亿参数，48kHz高保真 |
| 数据库 | SQLite | 轻量级关系型数据库 |
| 任务队列 | Celery (可选) | 异步任务处理 |

### 2.2 风格特征提取模块

#### 2.2.1 功能描述
- 音频加载与预处理
- 声学特征提取（调式、BPM、和声进行）
- 频谱特征提取（CQT频谱图、MFCC）
- 风格向量生成

#### 2.2.2 特征提取流程
```
音频输入 → 重采样(22050Hz) → 归一化 → 特征提取 → 风格向量
                                       ↓

```


## 3. 数据库设计

### 3.1 SQLite 数据表设计

```sql
-- 歌曲信息表
CREATE TABLE songs (
    id TEXT PRIMARY KEY,
    title TEXT NOT NULL,
    artist TEXT NOT NULL,
    album TEXT,
    cover_url TEXT,
    duration INTEGER,
    genre TEXT,
    release_date TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 风格特征表
CREATE TABLE style_features (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    song_id TEXT NOT NULL,
    bpm REAL,
    key TEXT,
    mode TEXT,
    style_embedding BLOB,  -- 存储numpy数组
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (song_id) REFERENCES songs(id)
);

-- 生成作品表
CREATE TABLE generated_songs (
    id TEXT PRIMARY KEY,
    reference_song_id TEXT,
    style_similarity REAL,
    originality_score REAL,
    audio_path TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (reference_song_id) REFERENCES songs(id)
);
```
