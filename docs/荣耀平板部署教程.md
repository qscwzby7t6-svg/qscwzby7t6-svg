# 🎵 荣耀平板 + Aid Learning 逐条部署教程

> **适用设备**: 荣耀平板（MagicPad 13、V7 Pro等）+ Aid Learning 应用  
> **系统版本**: Android 11+  
> **作者**: AI音乐创作平台开发团队

---

## 📋 准备工作

1. 在荣耀平板上打开 **Aid Learning** 应用
2. 等待系统启动完成（约1-2分钟）
3. 点击底部菜单栏的 **终端** 图标打开命令行

---

## ⚠️ 关于 sudo 命令

**在 Aid Learning 中，大多数操作需要使用 `sudo`**，因为默认用户可能没有足够的权限安装系统包。

| 场景 | 是否需要 sudo | 说明 |
|------|--------------|------|
| `apt install` | ✅ 需要 | 安装系统级软件包 |
| `apt update/upgrade` | ✅ 需要 | 更新系统 |
| `pip install` | ❌ 不需要 | 安装用户级Python包 |
| `git clone` | ❌ 不需要 | 克隆代码 |
| `mkdir/rm` | ❌ 不需要（用户目录内） | 创建/删除文件 |

## 🚀 逐条部署步骤

### 步骤1: 更新系统（有风险）

**⚠️ 风险提示**: 系统更新可能导致环境不稳定或存储空间不足

```bash
# 主方案：更新系统（推荐）
sudo apt update && sudo apt upgrade -y
```

**🔄 备用方案**（更新失败或速度慢时）:

```bash
# 方案A: 修复损坏的包后再更新
sudo apt update --fix-missing && sudo apt upgrade -y

# 方案B: 跳过某些大更新
sudo apt update
sudo apt upgrade --fix-broken -y

# 方案C: 如果更新卡住，按 Ctrl+C 中断，跳过此步骤继续
```

**✅ 验证**: 命令执行完成后没有报错

---

### 步骤2: 安装系统依赖（关键！）

**⚠️ 风险提示**: 安装包较多，可能需要较长时间和较多存储空间

```bash
# 主方案：一次性安装所有系统依赖
sudo apt install git wget build-essential gcc g++ gfortran cmake python3-dev python3-pip python3-venv libsndfile-dev ffmpeg libblas-dev liblapack-dev libatlas-base-dev -y
```

**🔄 备用方案**（某些包名不存在时）:

```bash
# 方案A: 使用替代包名
sudo apt install git wget build-essential gcc g++ cmake python3-dev python3-pip python3-venv libsndfile1-dev ffmpeg libblas-dev liblapack-dev -y

# 方案B: 分步骤安装（逐个确认）
sudo apt install git wget -y
sudo apt install build-essential gcc g++ gfortran cmake -y
sudo apt install python3-dev python3-pip python3-venv -y
sudo apt install libsndfile-dev ffmpeg -y
sudo apt install libblas-dev liblapack-dev libatlas-base-dev -y
```

**✅ 验证**: 所有包安装完成，没有"Unable to locate package"错误

---

### 步骤3: 克隆项目

**⚠️ 风险提示**: 网络问题可能导致克隆失败

```bash
# 主方案：克隆项目
git clone https://github.com/qscwzby7t6-svg/qscwzby7t6-svg.git
```

**🔄 备用方案**（网络问题时）:

```bash
# 方案A: 使用代理加速（如果配置了代理）
git clone https://ghproxy.com/https://github.com/qscwzby7t6-svg/qscwzby7t6-svg.git

# 方案B: 手动下载 ZIP 包（在浏览器中打开仓库下载）
# 1. 在 Aid Learning 内置浏览器打开 https://github.com/qscwzby7t6-svg/qscwzby7t6-svg
# 2. 点击 "Code" -> "Download ZIP"
# 3. 解压到 ~/projects 目录
unzip qscwzby7t6-svg-main.zip -d qscwzby7t6-svg

# 方案C: 检查网络连接
ping github.com  # 如果失败，检查网络设置
```

**✅ 验证**: 执行后出现 `Cloning into 'qscwzby7t6-svg'...` 提示，最终完成

---

### 步骤4: 进入项目目录并创建虚拟环境

```bash
# 进入项目目录
cd qscwzby7t6-svg

# 创建虚拟环境（不需要sudo）
python3 -m venv venv
```

**🔄 备用方案**（缺少 venv 模块时）:

```bash
# 方案A: 安装 python3-venv
sudo apt install python3-venv -y
python3 -m venv venv

# 方案B: 使用 pip 安装 virtualenv
pip install virtualenv --user
python3 -m virtualenv venv
```

**✅ 验证**: 当前目录下出现 `venv` 目录

---

### 步骤5: 激活虚拟环境

```bash
# 激活虚拟环境（不需要sudo）
source venv/bin/activate
```

**✅ 验证**: 命令行提示符前出现 `(venv)` 标记

---

### 步骤6: 安装 Python 依赖（分步安装）

**⚠️ 风险提示**: 安装时间较长，可能需要10-30分钟，确保电量充足

#### 步骤6.1: 升级 pip 并设置镜像

```bash
# 升级 pip（不需要sudo）
pip install --upgrade pip

# 设置国内镜像加速（强烈推荐）
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

#### 步骤6.2: 安装基础包（先装预编译版本）

```bash
# 安装 numpy 和 scipy（编译耗时，先装预编译版本）
pip install numpy==1.24.0 scipy==1.11.0
```

**🔄 备用方案**（编译失败时）:

```bash
# 方案A: 增加超时时间
pip install numpy==1.24.0 scipy==1.11.0 --default-timeout=300

# 方案B: 使用 conda 安装（如果安装了 conda）
conda install numpy=1.24.0 scipy=1.11.0 -y
```

#### 步骤6.3: 安装 PyTorch CPU 版本

```bash
# 安装 PyTorch CPU 版本（平板通常没有GPU）
pip install torch==2.0.0 torchaudio==2.0.0 --index-url https://download.pytorch.org/whl/cpu
```

**🔄 备用方案**（下载失败时）:

```bash
# 方案A: 使用国内镜像
pip install torch==2.0.0 -i https://pypi.tuna.tsinghua.edu.cn/simple

# 方案B: 跳过 PyTorch（仅使用基础功能）
# pip install torch==2.0.0 --no-deps
```

#### 步骤6.4: 安装其他依赖

```bash
# 安装 requirements.txt 中的其他依赖
pip install -r requirements.txt
```

**🔄 备用方案**（版本冲突时）:

```bash
# 方案A: 逐个安装，跳过冲突的包
pip install streamlit==1.28.0 fastapi==0.104.0 uvicorn[standard]==0.24.0
pip install librosa==0.10.0 soundfile==0.12.0 pydub==0.25.1
pip install pydantic==2.5.0 pydantic-settings==2.0.0
pip install pytest==7.4.0 pytest-asyncio==0.21.0

# 方案B: 使用 --force-reinstall 重新安装
pip install -r requirements.txt --force-reinstall
```

**✅ 验证**: 执行命令 `python -c "import numpy; import scipy; import torch; import librosa; print('依赖安装成功！')"` 没有报错

### 步骤9: 启动应用

#### 方式A: 前台运行（适合调试）

```bash
# 启动应用（不需要sudo）
streamlit run app.py --server.port=8501
```

#### 方式B: 后台运行（推荐，关闭终端不影响）

```bash
# 后台运行并保存日志
nohup streamlit run app.py --server.port=8501 --server.headless=true > app.log 2>&1 &

# 查看进程是否启动
ps aux | grep streamlit
```

**✅ 验证**: 输出中包含 `You can now view your Streamlit app in your browser.`

#### 方式A: 平板本地访问

1. 打开 Aid Learning 内置浏览器

## 🛠️ 常见问题快速解决方案

### Q1: 提示 "Permission denied"

```bash
# 使用 sudo 重新执行
sudo [命令]

# 或修改目录权限
chmod -R 775 ~/projects
```

### Q2: pip install 超时

```bash
# 设置超时时间
pip install [包名] --default-timeout=300

# 或使用国内镜像
pip install [包名] -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q3: 存储空间不足

```bash
# 清理 apt 缓存
sudo apt clean
sudo apt autoremove -y

# 清理 pip 缓存
pip cache purge

# 删除临时文件
rm -rf /tmp/*
```

### Q4: 内存不足（安装时被杀）

```bash
# 创建 swap 文件（增加虚拟内存）
sudo dd if=/dev/zero of=/swapfile bs=1M count=2048
sudo mkswap /swapfile
sudo swapon /swapfile

# 限制线程数
export OMP_NUM_THREADS=2
export MKL_NUM_THREADS=2
```

### Q5: 端口被占用

```bash
# 查看端口占用
lsof -i :8501

# 杀死占用进程
kill -9 <PID>

# 使用其他端口
streamlit run app.py --server.port=8502
```

### Q6: 应用启动后无法访问

```bash
# 检查防火墙设置
sudo ufw status

# 如果启用了防火墙，允许端口
sudo ufw allow 8501/tcp

# 检查网络连接
ping localhost
```

## 📝 停止应用

```bash
# 方法A: 如果是前台运行，按 Ctrl+C

# 方法B: 如果是后台运行
ps aux | grep streamlit  # 找到进程ID
kill -9 <进程ID>

# 方法C: 停止所有 streamlit 进程
pkill -f streamlit
```
